• Current behavior in your app is:

  1. ROI is treated as a yearly percentage.
  2. returnPeriod decides how often that yearly return is paid out.
  3. Each payout = (investment * ROI%) / period_divisor.

  So yes, it is the second one you said: yearly ROI, then split by selected processing period.

  Period divisors now:

  - annual: divide by 1 (full yearly return once per year)
  - semi-annual: divide by 2
  - quarterly: divide by 4
  - testing: divide by 365*8 (every 3 hours)

  Example (investment=10000, ROI=12%):

  - Yearly return = 1200
  - Quarterly payout = 300 each run
  - Semi-annual payout = 600 each run
  - Annual payout = 1200 each run
  - Testing payout = 1200/2920 ≈ 0.41 every 3 hours

  Also important: payouts are triggered by global cron schedule, not strictly per-user assignment anniversary.
